## Findings considered and rejected

- **Chat history re-render** (`chat.astro:100-101` rebuilds the whole history
  per turn): not worth doing — single-user, short sessions; O(n²) is
  irrelevant at this scale. Revisit only if long chats become common.
- **`kb restart` TOCTOU** (`cli._pids_on_port`): single-user; concurrent
  restart invocations racing is implausible. Low value.
- **Full bearer-token rearchitecture** (per-device tokens, HttpOnly cookies):
  L-effort and the single-user-on-Tailscale model makes the current shared
  bearer an acceptable documented tradeoff. Kept as a low-priority direction
  only; the immediate hardening is plan 009 (CSP) + plan 019 (health).
- **`cognee==0.3.*` glob pin**: not a finding (documented deliberate pin), but
  plan 012 notes the drift risk (`uv lock --upgrade` can pass 0.3.9 silently)
  and recommends a CI version assertion or pinning `==0.3.9`.
